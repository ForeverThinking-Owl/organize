# 工具、人与组织——实践的第一性原理

---

## 一、实践的底层问题：可利用性

生命存在于一个需要持续转化的世界。在热力学意义上，世界趋向扩散、耗散和均质化；如果缺少持续的能量输入、结构维持和差异生成，系统终将走向无差异的平衡。对生命而言，这种平衡意味着**可利用性的消失**。

生命作为开放系统，必须不断从环境中摄取能量、物质和信息，并通过代谢、排泄、运动、感知、选择和反馈，把外部对象转化为自身可以利用的结构。

> 牛吃草就是一个清楚的例子。草并没有直接以牛可以吸收的蛋白质形式存在。反刍动物依赖瘤胃中的微生物发酵饲料，分解植物纤维，产生可利用的能量和营养形式。牛的生命活动依赖一个由草、瘤胃、微生物、发酵过程和消化系统共同构成的转化结构。

这个例子提示我们，生命活动的核心是**转化环境与自身之间的关系**。

**实践，是生命为了维持或扩展自身的有序结构，借助某种中介，把环境中尚未直接可用的对象转化为可利用之物的过程。**

这里的"可利用"包括：可利用的能量、物质、信息、路径、工具、记忆和协作。混乱本身并非核心问题，核心问题是可利用性的丧失。生命无法让整个世界有序，却可以在局部持续制造可利用性。

> **实践的第一性含义，是可利用性的生产。**

---

## 二、工具：被稳定下来的转化方式

在人类这里，转化获得了新的形式。人类依靠身体结构、生理机制和共生系统转化环境，同时把某些转化方式外化为工具：

- **石器**把切割、敲击、剥离稳定下来
- **火**把加热、照明、防御和聚集稳定下来
- **语言**把经验传递稳定下来
- **文字**把记忆稳定下来
- **账簿**把核算与追责稳定下来
- **机器**把力量稳定下来
- **计算机**把信息处理稳定下来

**工具是被外化、稳定化、可重复执行的转化方式。**

一块石头只有进入切割、敲击或制造的实践，才成为工具。一个接口只有进入订单处理、库存调度、客户服务或财务核算，才成为工具。工具通过实践获得意义，它在实践中稳定了一种可重复的转化方式。

早期石器的重要性并不只在于物体形态，而在于它们把敲击、切割、剥离、加工等实践方式外化出来，使这些方式可以被重复、学习、传递和改进。工具改变了实践的形式。生命依靠身体结构和即时行为转化环境；工具则让转化方式脱离当下身体，成为可保存、可传授、可组合、可扩展的外部形式。

在现代系统中：

| 工具类型 | 进入实践才成为工具 |
|----------|-------------------|
| API | 进入某种社会生产实践 |
| RPA | 进入老系统录入、查询、提交、回写的实践 |
| MCP Tool | 进入受规则、对象和责任约束的行动链条 |
| 软件界面 | 进入人的判断和操作流程 |
| 自动化脚本 | 进入可重复、可审计的生产链路 |

> **工具的本质，是稳定下来的转化方式。**

## 三、技能：工具的编排

工具本身尚未构成技能。

- 会拿刀，距离会做饭仍有一段**实践距离**
- 会打开系统，距离处理客户投诉仍有一段**判断距离**
- 会调用接口，距离完成退款审查仍有一段**责任距离**
- 会触发自动化，距离完成生产实践仍有一段**协同距离**

技能高于工具清单，它表现为人对工具的掌握、选择、组合和判断性使用。

> **技能，是工具的编排。**

一个厨师的技能，体现在他知道什么时候用刀、怎样用火、如何处理食材、如何控制顺序和时间。一个医生的技能，体现在他能在具体病情中判断如何使用问诊、检验、影像和药物。一个工程师的技能，体现在他能把代码仓库、日志系统、监控面板和部署系统编排成诊断、修改、测试和发布的实践过程。

工具外化实践方式，技能则把工具重新内化为人的实践能力。工具让某种转化方式可以保存，技能让这种转化方式可以被人承担。工具是可执行的实践方式，技能是人对这些实践方式的组织化使用。

这也解释了工具调用与实践之间的差异：一个系统可以调用很多接口，但如果它缺少对**对象、目的、顺序、边界、责任和反馈**的判断，它只是触发动作。技能意味着：

- 工具之间有**秩序**
- 工具使用有**目的**
- 工具选择有**判断**
- 工具结果有**反馈**

---

## 四、人：技能、判断与责任的承载者

人是技能、判断、目的和责任的承载者。

人在实践中并非抽象存在，而是带着某种技能结构出现。有人擅长制造，有人擅长照料，有人擅长计算，有人擅长沟通，有人擅长判断风险，有人擅长修复系统，有人擅长设计规则，有人擅长协调他人。人的实践能力，既来自对工具的掌握，也来自对情境的判断。

同样一把刀，可以用于烹饪，也可以用于修理，也可以用于伤害。**工具本身无法决定实践目的。**工具必须被放入目的、规则、边界和责任结构中。人的意义，在于把工具、技能、目的和责任连接起来。

实践中的人至少承担三件事：

1. **判断目的**：为什么要做这件事
2. **选择方式**：用什么工具、以什么顺序、在什么边界内做
3. **承担后果**：如果结果错误、越界、失败或伤害他人，责任如何归属

> 一个模型可以调用工具，一个脚本可以执行步骤，一个系统可以写入数据，但目的、对象、规则、责任和反馈决定了它能否进入实践。人把工具、技能和责任放入同一个结构中，使实践获得方向和后果。

---

## 五、协同：复杂实践超出单个人的技能边界

当某一种实践需要超过单个人的技能边界，不同技能就必须协同。

- 狩猎需要观察、追踪、制造、围捕和分配
- 医疗需要诊断、护理、检验、手术、药剂和管理
- 制造需要设计、采购、加工、质检、仓储和交付
- 客户服务需要沟通、查询、判断、售后、财务、技术和反馈

足够复杂的实践，都需要多种技能在同一个对象周围形成协同。

协同意味着不同技能围绕共同对象发生关系：有人负责识别对象，有人负责处理对象，有人负责提供工具，有人负责判断风险，有人负责执行写入，有人负责反馈结果，有人负责修正规则。

> 复杂实践之所以复杂，并非因为步骤多，而是因为不同技能之间必须被安排到一个**可持续的关系**中。

在这里，稳定结构开始出现。复杂实践要求不同技能在规则和责任中协同，协同一旦稳定下来，就形成更高阶的实践结构。

---

## 六、组织：协同的稳定结构

**组织是人的技能围绕复杂实践需要形成的稳定协同结构。**

- 一个狩猎队伍，是观察、追踪、制造、围捕、分配等技能的协作结构
- 一个医院，是诊断、护理、检验、手术、药剂、管理等技能的协作结构
- 一个工厂，是设计、采购、制造、质检、仓储、销售等技能的协作结构
- 一个企业，是销售、客服、财务、研发、运营、交付等技能围绕社会生产需要形成的稳定结构

组织的意义在于实践。组织为了持续满足某种社会实践需要而形成。它保存工具，分配技能，定义角色，限制边界，建立责任，沉淀记忆，并在反馈中修正自身。

组织至少包含六个要素：

| 要素 | 作用 |
|------|------|
| **对象** | 提供实践方向 |
| **技能** | 提供行动能力 |
| **工具** | 提供转化方式 |
| **规则** | 限定边界 |
| **责任** | 承接后果 |
| **记忆** | 保存经验并修正未来 |

因此，实践是第一概念。工具是实践方式的外化，技能是工具的编排，协同是多种技能共同回应复杂实践，组织则是协同的稳定结构。

> **组织是为了满足某种复杂实践需要，对人、技能、工具、规则、责任和记忆进行稳定编排的形式。**

---

## 七、AI：实践编排方式的变化

AI 是这条链条中的新变量。它的意义，在于**改变工具、技能和协同之间的编排方式**。

过去，工具主要由人手动选择和操作。人识别对象，人选择工具，人组合步骤，人判断结果，人推动下一步。软件系统大多等待人输入，传统 RPA 主要在固定脚本中模拟人的点击和录入。

AI，尤其是大模型，使工具进入新的阶段。它可以理解语言，识别对象，解释上下文，选择工具，组合步骤，调用系统，读取反馈，并在必要时请求治理。工具的选择、组合和执行开始可以被语言、情境和规则驱动，技能的部分结构开始可以被程式化，协同的部分调度开始可以自动化。

**AI 的意义在于改变实践的调度方式。**它使实践的 5W1H 开始可以被处理：

| 维度 | 含义 |
|------|------|
| **何时** (When) | 什么时候启动、暂停、升级或终止 |
| **何地** (Where) | 在哪个系统、场景、单元或上下文中发生 |
| **何事** (What) | 这到底是什么对象和问题 |
| **何因** (Why) | 为什么在此情境下需要这样行动 |
| **何人** (Who) | 由哪个角色、哪种技能、哪个协同结构承担 |
| **如何** (How) | 调用哪些工具，以什么顺序、什么边界执行 |

**AI 获得的是实践调度权。实践主权仍然属于人和组织。**

实践主权包括：目标、边界、风险、例外和责任。人决定什么目标值得追求，什么边界需要守住，什么风险必须确认，什么例外需要裁决，什么结果需要承担责任。

人的位置因此发生上移：

| 过去（执行者） | 未来（治理者） |
|---------------|---------------|
| 查询者 | 目标设定者 |
| 录入者 | 边界制定者 |
| 催办者 | 例外裁决者 |
| 复制者 | 风险审批者 |
| 执行者 | 规则修正者 |
| — | 最终责任者 |

## 八、从协同到自主运行

当工具的可执行性提高，技能的程式化程度提高，协同的调度可以被语言、情境和规则驱动，组织就可能走向一种新的形态：**自主运行的组织形式**。

这里的"自主运行"指向一种实践结构：组织围绕某种实践需要，自动编排工具、技能与人，并在治理和反馈中持续运行。它依然包含人的目标、边界、责任和判断；同时，常规实践的调度和执行逐渐交给由工具、技能和 AI 共同构成的运行结构。

这种组织形式的基本结构可以表述为：

```
实践需要 → 工具
工具编排 → 技能
技能承担 → 人
人的技能协同 → 组织
组织运行 → 社会实践
实践反馈 → 新工具、新技能、新协同
```

在这种结构中，AI 参与实践调度。它帮助判断什么时候启动实践，在哪里发生实践，处理什么对象，为何如此行动，由谁承担实践，如何组合工具完成实践。人保留实践主权：定义目标、设置边界、审批风险、处理例外、承担责任，并修正下一次实践的规则。

> **自主运行的组织，是人的技能、工具和判断被重新编排后，组织能够持续满足复杂实践需要的运行形态。**

AI 时代真正值得讨论的问题：

- 工具如何被编排为技能
- 技能如何由人承担
- 人的技能如何被组织起来
- 组织如何满足社会实践的需要
- AI 如何参与实践调度
- 人又如何保留实践主权

---

## 九、结语：工具、人与组织

世界天然趋向耗散，生命必须维持可利用性。

| 概念 | 本质 |
|------|------|
| **实践** | 把尚未直接可用之物转化为可利用之物 |
| **工具** | 被外化、稳定化、可重复执行的转化方式 |
| **技能** | 人对工具的掌握、选择、组合和判断性使用 |
| **协同** | 多种技能共同回应复杂实践需要 |
| **组织** | 协同的稳定结构 |

工具是可执行的实践方式。技能是工具的编排。人是技能、判断与责任的承载者。组织是人的技能围绕社会实践需要形成的协作结构。

AI 的意义，在于改变工具、技能和协同的编排方式。它使实践的 5W1H 可以被语言、情境和规则调度，使更多常规实践可以自动运行。

> **AI 获得实践调度权，人保留实践主权。**实践的目的、边界、例外和责任，仍然需要由人和组织承担。

最终，社会需要能够围绕实践需要，把工具、技能与人组织起来的运行结构。

**这就是工具、人与组织的第一原理。**
